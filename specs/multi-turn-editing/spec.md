# 多轮对话编辑功能 Spec

## Why
用户在使用提示词生成功能时，AI给出的二级提示词可能需要调整。用户需要能够对提示词进行修改、补充或重新生成，而无需从头开始输入完整需求。

## What Changes
- 新增暗号识别：`c`（单字母，位于用户输入结尾）触发提示词整改模式
- 新增结束暗号：`end` 或 `结束` 清除上下文记忆，开始新的对话
- 支持多种整改类型：重新设计、替换元素、修改场景、修改角色行为等
- 上下文记忆：在多轮对话中保持对原始需求的理解

## Impact
- Affected specs: activation-code, prompt-levels, shot-composition
- Affected code: `.trae/skills/prompt-crafter/SKILL.md`

## ADDED Requirements

### Requirement: 整改暗号识别
系统 SHALL 在用户输入结尾检测到单独字母 `c` 时，识别为整改需求，进入提示词编辑模式。

#### Scenario: 整改暗号触发
- **WHEN** 用户输入以单个字母 `c` 结尾（前面有空格）
- **THEN** 系统进入整改编辑模式，分析用户描述的整改需求

#### Scenario: 整改需求表达
- **WHEN** 用户描述整改需求
- **THEN** 整改需求表达方式灵活多样，不限于特定关键词。系统根据用户描述的修改意图，结合当前上下文记忆，自动解析并应用修改

**常见整改表达示例**（仅供参考，不限于这些）:
- `重新设计三个镜头 c` → 重新设计分镜头数量
- `景别改为中景 c` → 修改镜头景别
- `角色的裤子换成长裤 c` → 替换服装描述
- `场景改为校园 c` → 修改场景环境
- `角色的行为改为滑滑板 c` → 更新角色动作
- `景别大一点 c` → 调整景别
- `角色换成男孩 c` → 替换主体描述

### Requirement: 结束暗号识别
系统 SHALL 在用户输入 `end` 或 `结束` 时，清空当前对话上下文记忆。

#### Scenario: 结束暗号触发
- **WHEN** 用户输入为 `end` 或 `结束`（忽略前后空格）
- **THEN** 系统清空上下文记忆，准备好接受新的提示词需求

### Requirement: 多轮对话上下文管理
系统 SHALL 在多轮对话中保持对原始提示词需求的理解，支持累积修改。

#### Scenario: 连续整改
- **WHEN** 用户连续输入多个整改需求（均以 `c` 结尾）
- **THEN** 系统依次应用每个整改需求，最终输出修改后的完整提示词

#### Scenario: 上下文保留
- **WHEN** 用户进行多次整改后输入新的原始需求
- **THEN** 系统在新需求前清空旧上下文，避免混淆

### Requirement: 整改输出格式
系统 SHALL 在整改模式下输出修改后的完整提示词，而非仅输出差异部分。

#### Scenario: 整改后输出
- **WHEN** 用户触发整改模式并成功应用修改
- **THEN** 输出完整的英文提示词 + 中文翻译，与首次生成格式一致

## MODIFIED Requirements

### Requirement: 暗号激活码扩展
原 activation-code spec 中规定的激活码从 `i`（图生图）、`v`（图生视频）扩展为：
- `i` → 图生图提示词生成
- `v` → 图生视频提示词生成
- `c` → 提示词整改编辑（新增）
- `end/结束` → 清空上下文（新增）

**Migration**: 原有 `i`/`v` 功能保持不变，新增 `c` 和 `end/结束` 的处理逻辑
